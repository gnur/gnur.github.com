DCOS

notendop

  • centos / rhel
  • veel componenten
  • bootstrap vanuit DCOS
  • werkt met VAMP
  • volwassen (?)

pluspunten

  • goede webinterface
  • goede logging opties
  • gebruikt healthchecks voor een rolling release
  • vamp ondersteuning

minpunten

  • complex
  • distributed state
  • veel componenten (zookeper, mesos, marathon, dns)
  • geen drain-mode om nodes weg te halen
  • geen "system" jobs (tasks die op elke node moeten draaien)
  • consul integratie is meh
  • door enorme overhead is het niet wenselijk om meerder clusters te draaien, waardoor consul ook unified moet worden
  • recovery is complex (niet via web ui, state uit zookeeper halen)

nomad

notendop

  • os onafhankelijk (bsd, windows, linux)
  • single binary
  • hashicorp

pluspunten

  • simpel
  • consul integratie
  • kan op debian draaien (meer interne kennis)
  • minder overhead (nomad servers/masters kunnen waarschijnlijk op micro of small draaien)
  • meerdere workloads mogelijk (binaries, jar, docker, rocket)
  • system jobs (jobs die op elke node moeten draaien)
  • node drainage is mogelijk (voor onderhoud of clean shutdown)

minpunten

  • relatief jong
  • unknown unknowns
  • werkt niet met vamp
  • rolling releases zijn (nog) niet op basis van consul healthchcecks
  • geen standaard web ui (en de beschikbare UI is read-only)

kubernetes

notendop

  • werkt op ubuntu
  • google
  • planet scale

pluspunten

  • google (15 jaar productie ervaring met containers)
  • werkt met vamp
  • waarschijnlijk heel goed

minpunten

  • niemand heeft er ervaring mee
  • extreem veel uitzoekwerk
